# Medical Architecture

`nexa_medical` verwaltet Verletzungen, Blutungen, Schmerz, Bewusstlosigkeit, Tod, Krankenhaus-Respawn, EMS-Behandlung, Medikamente, Stabilisierung, Tragen als Workflow-Grenze und Medical Reports.

Der Client meldet nur Beobachtungen. State, Tod, Respawn, Behandlung und Reports sind serverseitig.

# Inventory Architecture

`nexa_inventory` ist das serverautoritative Inventory-Grundsystem. Es verwaltet Character-Inventare, technische Container, temporare Drops und zukuenftige Storage-Typen. Standard fuer Character: 30 Slots, 30.000 Gramm, 5 Quickslots.

Die Resource bietet keine UI und keine Itembenutzung. Sie stellt sichere Exports, Nexa-Callbacks, interne Events, Audit und Runtime-Tests bereit.

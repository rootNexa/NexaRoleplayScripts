# nexa_maps

Zentrale Map- und MLO-Datei-Registry fuer Nexa Roleplay.

Phase 10E umfasst nur:

- Map-/MLO-Datei-Registry
- Lade-/Aktivierungsstatus
- Resource-Zuordnung
- Map-Kategorien
- Umgebungskonfiguration
- Dokumentation fuer spaetere Add-on-Maps und MLOs

Nicht enthalten:

- echte Map-Dateien
- MLO-Erstellung
- 3D-Modelle
- OpenIV-/CodeWalker-Assets
- neue Gameplay-Systeme
- echte Marken oder echte Behoerden

`nexa_maps` verwaltet nur Registry und Dokumentation. Spaetere Add-on-Maps muessen ueber neue Registry-Eintraege ohne Breaking Change ergaenzbar sein.

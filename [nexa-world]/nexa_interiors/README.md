# nexa_interiors

Zentrale MLO- und Interior-Registry fuer Nexa Roleplay.

Phase 10D umfasst nur:

- Interior-Konfiguration
- MLO-/Interior-Registry
- Entry-/Exit-Punkte als Registry-Daten
- Interior-Permissions
- Doorlock-Vorbereitung als Registry-Feld
- Storage-/Garage-/Faction-Verknuepfungen als Registry-Felder
- serverseitige Zugriffvalidierung
- Audit und Logging

Nicht enthalten:

- echte MLO-Dateien
- 3D-Modelle
- Doorlock-Vollintegration
- Teleport-Logik
- neue Gameplay-Systeme
- echte Marken oder echte Behoerden

Der Client entscheidet keinen Interior-Zugriff final. Spätere MLOs muessen ueber die Registry ohne Breaking Change ergaenzbar sein.
